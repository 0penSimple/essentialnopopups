Good to know

  • Everything happens in your browser — your files are never uploaded anywhere.
  • Supported formats: plain text (.txt), Markdown (.md), and Word documents (.docx). Word documents are returned with all formatting fully preserved — bold, italic, font sizes, headings and all.
  • PDF files cannot be edited while preserving their layout. When you include a PDF, the text is extracted and returned as a plain .txt file instead.
  • Case sensitive means the search will match the exact capitalisation you type. For example, searching "Hello" will not find "hello" or "HELLO". Leave it unchecked to find the word regardless of how it is capitalised — this is the default and works for most cases.
  • Text in page headers and footers (the repeating content at the very top or bottom of every printed page) is not searched — only the main document body is included.
  • In rare cases, Word may have internally split a word across multiple formatting fragments — for example if part of a word was ever formatted differently. If you expect matches but see zero for a .docx file, try copying the content into a fresh document and re-uploading.
  • Rules are applied in a single pass — if rule 1 changes "cat" to "dog" and rule 2 searches for "dog", the newly created word will not be caught by rule 2. This is intentional and prevents rules from interfering with each other.
  • Only files with at least one match are included in the download. Files with no matches are skipped — you will see a summary after downloading.
  • Select which files to include in the replacement using the checkboxes. Unchecked files are searched but not modified.
